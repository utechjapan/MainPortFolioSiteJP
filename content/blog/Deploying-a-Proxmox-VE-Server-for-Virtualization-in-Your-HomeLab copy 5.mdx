## Proxmox VEとは？

Proxmox Virtual Environment（Proxmox VE）は、仮想マシン（VM）とコンテナ（LXC）の管理が可能な、オープンソースの仮想化プラットフォームです。ホームラボにおいて、シンプルかつ効率的に環境構築を行えるため、実験や学習に最適です。本ガイドでは、Proxmox VEサーバーのインストールから基本的な操作までを網羅的に説明します。

---

## 準備するもの

- 専用マシン（推奨要件：64ビットプロセッサ、8GB以上のRAM、64GB以上のストレージ）
- [Proxmox公式サイト](https://www.proxmox.com/ja/downloads)でダウンロードしたISOファイル
- ブート可能なUSBドライブ（8GB以上推奨）
- 安定したインターネット環境

---

## ステップ 1: インストールメディア作成

1. Proxmox VEのISOファイルをダウンロードします。
   ![Proxmox Interface](/images/downloadsproxmox.png)
2. [Rufus](https://rufus.ie/)や [balenaEtcher](https://www.balena.io/etcher/)を使用し、USBドライブをブータブルにします。

---

## ステップ 2: Proxmox VEのインストール

1. 作成したUSBでサーバーを起動します。
2. インストールウィザードに従い、以下の設定を行います：
   - ライセンス同意
   - インストールディスク選択
   - rootパスワードおよびメールアドレスの入力
   - 固定IPなどのネットワーク設定
3. インストール後、USBを抜き再起動します。

---

## ステップ 3: Webインターフェースへのアクセス

1. 他のデバイスからウェブブラウザを開き、`https://<ProxmoxサーバーIP>:8006` にアクセスします。
2. 設定したrootユーザーでログインします。

---

## ステップ 4: Proxmox VEの初期設定と更新

1. システムを最新化します。
```bash
apt update && apt full-upgrade -y
```
2. エンタープライズリポジトリを無効化します。
```bash
sed -i 's|deb https://enterprise.proxmox.com/debian|#deb https://enterprise.proxmox.com/debian|' /etc/apt/sources.list.d/pve-enterprise.list
```
3. コミュニティリポジトリを追加します。
```bash
echo "deb http://download.proxmox.com/debian/pve bullseye pve-no-subscription" | sudo tee /etc/apt/sources.list.d/pve-no-subscription.list
```
4. 再度システムを更新します。
```bash
apt update && apt full-upgrade -y
```

---

## ステップ 5: ストレージの設定

Webインターフェースの **Datacenter** からストレージ（ローカルまたはネットワーク）を追加します。

---

## ステップ 6: 仮想マシン（VM）の作成

1. ISOイメージをアップロードします。
   - **Datacenter** → **Storage** → **ISO Images** → **Upload**
     ![ISO Upload Location](/images/proxmoxiso.png)
2. VMを作成します。
   - **Create VM** をクリックして、リソースを設定します。
3. VMを起動し、OSをインストールします。

---

## ステップ 7: コンテナの利用（オプション）

1. コンテナテンプレートをダウンロードします。
   - **Datacenter** → **Storage** → **CT Templates** → **Download**
2. コンテナを作成します。
   - **Create CT** をクリックして設定します。

---

## ステップ 8: バックアップとリストア

1. VMやコンテナをバックアップします。
   - **Datacenter** → **Backup** → **Add** で設定
2. 必要に応じてリストアします。

---

## よくある質問

- **仮想マシンとコンテナの違いは？**
  - 仮想マシンは完全なOSを持ち、コンテナはホストのカーネルを共有するため軽量です。

- **Raspberry Piで動作する？**
  - いいえ、Proxmox VEはx86_64アーキテクチャ専用です。

- **GPUパススルー対応は？**
  - はい、対応しています。

- **仮想マシンへのアクセス方法は？**
  - Webコンソール、SSH、RDP、VNCなどを利用可能です。

- **Proxmoxを仮想環境で試すことは可能？**
  - 可能ですが、ネスト仮想化のサポートが必要です。
